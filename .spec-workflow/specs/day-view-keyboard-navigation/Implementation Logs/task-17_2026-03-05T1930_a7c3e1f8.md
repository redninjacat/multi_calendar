## Artifacts

### Production code changes (mcal_day_view.dart)

#### `MCalDayView.debugAnnouncementLog`
- **Purpose:** `@visibleForTesting` static `List<String>?` that, when non-null, captures all screen reader announcement strings. Allows widget tests to verify announcements without intercepting platform channels.

#### `_announceScreenReader` updated
- **Purpose:** Now writes to `debugAnnouncementLog` before sending the platform announcement. No behavioral change in production (field is always null).

#### Deprecated `SemanticsService.announce` removal
- **Purpose:** Replaced 14 direct `SemanticsService.announce(message, TextDirection.ltr)` calls across Navigation Mode, Event Mode, Move Mode, and Resize Mode handlers with `_announceScreenReader(context, message)` calls. This eliminates all deprecated API usage and ensures consistent announcement routing through a single entry point.

#### Missing Escape→Navigation Mode announcement
- **Purpose:** The top-level Escape handler in `_handleKeyEvent` cleared Event Mode state but did not announce "Navigation mode." Added `_announceScreenReader(context, l10n.announcementDayNavigationMode)` to match the fallback handler's behavior.

### Test groups (16 tests)

- **Navigation Mode** (4): ↑ from slot 0 → all-day section, ↓ from all-day → time grid, A key → all-day, T key → time grid
- **Event Mode** (6): Enter → event count, Escape → navigation mode, D → deletion, M → move mode, R → resize mode, X → conversion
- **Move Mode** (1): Escape → cancellation
- **Resize Mode** (4): Escape → cancellation, S → start edge, E → end edge, M → move mode
